Разработай Minecraft Launcher для моего сервера.

Основная идея:
лаунчер НЕ должен полностью перезаписывать папку mods. Игрок может добавлять свои моды самостоятельно. Лаунчер должен принудительно скачивать, обновлять и восстанавливать только обязательные файлы, которые указаны на сервере в manifest.json.

Minecraft:
- версия Minecraft всегда фиксированная;
- версия задаётся на сервере в manifest.json;
- loader тоже задаётся на сервере: Fabric / Forge / NeoForge;
- лаунчер должен запускать именно указанную сервером версию.

Файлы и обновления:
- лаунчер читает manifest.json с сервера;
- в manifest.json есть список обязательных файлов;
- каждый файл имеет путь, url, размер, sha256, категорию и флаг required;
- лаунчер проверяет только обязательные файлы;
- если обязательный файл отсутствует, повреждён или устарел, он скачивается заново;
- лаунчер НЕ удаляет пользовательские моды;
- лаунчер НЕ трогает файлы, которых нет в списке обязательных;
- должна быть кнопка “Проверить файлы” / “Repair”;
- должна быть история обновлений / changelog.

Настройки:
- раздел “Настройки”;
- выбор папки установки игры;
- возможность изменить папку, где будут храниться файлы лаунчера и сборки;
- переключатель “Шейдеры”;
- если игрок включает “Шейдеры”, лаунчер дополнительно скачивает shaderpacks и нужные файлы, указанные в manifest.json как optionalShaders;
- если игрок выключает “Шейдеры”, лаунчер не должен удалять пользовательские шейдеры, но может отключить автодокачку серверных shaderpacks;
- настройки должны сохраняться локально.

Карта:
- добавить раздел “Карта”;
- карта сервера работает через BlueMap;
- в manifest.json или настройках сервера должна быть ссылка на BlueMap;
- по возможности встроить BlueMap прямо в лаунчер через WebView;
- если встроить нельзя, добавить кнопку “Открыть карту в браузере”.

Интерфейс:
- главный экран с названием сервера, статусом сборки и кнопкой “Играть”;
- индикатор проверки и загрузки файлов;
- отображение текущей версии сборки;
- раздел “Новости”;
- раздел “Моды” со списком обязательных файлов и их статусом;
- раздел “Карта” с BlueMap;
- раздел “Настройки”;
- понятные сообщения об ошибках.

Запуск игры:
- перед запуском лаунчер проверяет обязательные файлы;
- если всё актуально, запускает Minecraft;
- если есть обновления, скачивает их;
- пользовательские моды должны оставаться на месте;
- желательно добавить настройку выделения RAM;
- желательно добавить выбор Java, но можно использовать встроенную/автоопределённую Java.

Пример manifest.json:
{
  "serverName": "My Minecraft Server",
  "packVersion": "1.0.0",
  "minecraftVersion": "1.20.1",
  "loader": "forge",
  "loaderVersion": "47.4.0",
  "blueMapUrl": "https://map.example.com",
  "requiredFiles": [
    {
      "path": "mods/create.jar",
      "url": "https://example.com/files/create.jar",
      "sha256": "HASH_HERE",
      "size": 12345678,
      "category": "mod",
      "required": true
    },
    {
      "path": "config/create-common.toml",

  "optionalShaders": [
